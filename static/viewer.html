    Why is this page empty?

    The proxy is a wire tap — it can only see API requests that physically travel through it. The Claude session you're currently chatting in was started without ANTHROPIC_BASE_URL set, so its requests go directly from claude → api.anthropic.com; they never touch port 9999. The proxy literally has no way to read them. Only Claude processes started through the proxy can be tapped.

    Fastest way to see something now

    Click the button below and the proxy will spawn a single claude -p "ping" subprocess with itself wired up as the upstream. That fires exactly one API request through the proxy, captures the full envelope (system prompt, every tool's verbatim description and input_schema, your message, Claude's response), and exits. Cost: about $0.001 in tokens, ~3 seconds.

    Or capture an interactive session yourself

    If you want to watch a real session live, start one through the proxy from any terminal:

    ANTHROPIC_BASE_URL=http://127.0.0.1:9999 claude

    Every request that session makes will appear here within ~2 seconds, with full envelope verbatim. Use this to verify what Claude Code is sending the model, audit tool descriptions against published docs, or watch how the prompt changes turn-by-turn.

    Still not seeing anything? Click to debug.
    • From any terminal: curl -s http://127.0.0.1:9999/api/taps should return [] (or a list once you've routed a request through). If it errors, the proxy isn't running.
    • In the new shell where you ran claude: echo $ANTHROPIC_BASE_URL should print http://127.0.0.1:9999. If it's empty, the env var didn't propagate.
    • Tail the proxy log for incoming POSTs: tail -f /tmp/monitor-claude-code-9999.log. You should see "POST /v1/messages HTTP/1.1" 200 lines as you chat.
    • List what the proxy has saved: ls ~/.claude/skills/monitor-claude-code/taps/.
